# FocusClip 개발 계획서

## 목표
FocusClip — 런처(FocusManager) + 클립보드(Clipboard-Manager)를 통합한 .NET 8 WPF 트레이 앱.
원본 FM/CM 동작을 충실히 재현하면서 기능을 점진적으로 개선한다.

---

## 우선순위 백로그

### ✅ 완료
- D1: 이미지 드래그 시 느림 해결 (파일 드롭 우선, 비트맵 폴백)
- D2: 텍스트 클립 드래그앤드롭 지원
- D3: 경로 팝업 드래그앤드롭 지원 (텍스트 전용)
- D4: 드래그 한자 깨짐 수정 (StringFormat 제거, SetText만 사용)

### 🔵 제안
- ✅ P001: 드래그 실패 시 토스트 알림 ("드롭 미지원 앱", Esc 취소는 제외)
- P002: 클립 팝업에서 텍스트 카드 우클릭 → "드래그용 임시 파일 생성" (txt 파일로 탐색기 드롭 지원)
- P003: 경로 팝업 — 로컬 경로 드래그 시 파일 드롭도 선택적으로 제공 (토글 설정)
- ✅ P004: 클립보드 항목 영구 저장 (clips.json, 텍스트/이미지/경로 모두)
- P005: 클립 팝업 검색 필드 (카드 수 많을 때 텍스트 빠른 검색)
- P006: 카드에 드래그 핸들(⠿) 아이콘 표시 — 드래그 가능함을 시각적으로 안내
- P007: 사용 빈도 기반 클립 자동 정렬 (자주 쓰는 항목 상단 고정 옵션)
- P008: 이미지 클립 카드에 썸네일 미리보기 (현재는 텍스트 레이블만 표시)
- P009: 클립 항목 최근 사용 시각 표시 (카드 hover 툴팁)
- P010: 클립 히스토리 JSON 내보내기/가져오기 (백업·기기 이전)
- P011: 트레이 우클릭 메뉴에 "클립 히스토리 전체 지우기" 항목 추가
- ✅ P012: 핀 고정 클립 항목 재시작 후에도 유지 (clips.json에 Pinned 포함)
- P013: 런처 도크 앱 아이콘 자동 갱신 (앱 업데이트로 아이콘 변경 시 캐시 재추출)
- P014: 클립 저장 용량 설정 (현재 고정 20개, 설정 창에서 변경 가능하게)
- P015: clips.json 손상 시 자동 백업 복구 (저장 전 .bak 유지) — ※ 부분 쓰기 손상은 원자적 저장으로 예방 완료, 남은 범위는 디스크 오류·외부 변조 등으로 이미 손상된 파일의 복구
- P016: 클립 팝업 핀 상태를 별도 색상으로 강조 (현재 핀/비핀 카드 구분이 약함)
- P017: 앱 종료 없이 설정에서 클립 히스토리 최대 개수 즉시 반영
- P018: 이미지 카드 로딩 중 스켈레톤 플레이스홀더 표시 (비동기 로드 중 빈 카드 방지)
- ❌ P019: (철회) 용량 초과 자동 파일 삭제 — 사용자가 media 폴더를 직접 관리(수동 삭제)하기로. 팝업은 최근 20개 뷰, 폴더 파일 수는 헤더에 표시
- 🔵 P020: media 폴더 고아 파일 정리 도우미 — clips.json이 참조하지 않는 본문 파일을 선택/일괄 삭제하는 버튼(완전 수동 삭제 방침 보완, 사용자가 원할 때만 실행)
- 🔵 P021: 헤더 폴더 표시에 총 용량(MB) 추가 — 파일 수 + 누적 용량으로 무한 증가 모니터링
- 🔵 P022: clips.json이 참조하는 본문 파일이 수동 삭제된 카드 처리 — 로드 시 "원본 없음" 배지 또는 자동 스킵(현재 텍스트는 빈칸/이미지는 무표시)
- 🔵 P023: 캡처 PNG 저장 시 무손실 압축 최적화 옵션 — 수동 관리로 파일이 누적되므로 디스크 절약
- 🔵 P024: 전역 미처리 예외 안전망 — AppDomain.UnhandledException / DispatcherUnhandledException / TaskScheduler.UnobservedTaskException 핸들러로 예기치 않은 예외에도 앱이 조용히 죽지 않고 로깅/복구(신호 스레드 크래시류 일반화)
